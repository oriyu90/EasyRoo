# EasyRoo — 使い方ガイド

**ローカルLLMで動く、Mac向けルーティーンAIアプリ**

新しいチャットを始める感覚でルーティーンを作り、手順書と注意書きを書いて、
決めた曜日・日・時刻にAIへ自動実行させます。
AIはターミナルを操作し、接続したMCPツールを使って作業を進めます。

- バージョン: **v1.0**
- 対応: macOS (Apple Silicon / arm64)
- ライセンス: MIT (Copyright © 2026 yuki_orita)

---

## 1. インストール

1. `EasyRoo-1.0.0-arm64.dmg` をダブルクリックして開きます。
2. **EasyRoo.app** を **Applications** フォルダにドラッグします。
3. アプリケーションフォルダから EasyRoo を起動します。

### 初回起動時の警告について

このアプリはApple Developer IDによる署名を行っていないため、初回起動時に
「開発元を確認できないため開けません」と表示されます。以下のいずれかで起動できます。

- **方法A**: EasyRoo.app を **右クリック → 「開く」** を選び、確認ダイアログで「開く」を押す
- **方法B**: 「システム設定 → プライバシーとセキュリティ」を開き、下部の
  「"EasyRoo"は開発元を確認できないため…」の横にある **「このまま開く」** を押す

一度許可すれば、次回からは通常どおりダブルクリックで起動できます。

## 2. 事前準備：LLMを接続する

EasyRoo は **OpenAI互換API** に対応しています。ローカルLLMでもクラウドでも構いません。

| 接続先 | ベースURL | APIキー |
| --- | --- | --- |
| LM Studio | `http://localhost:1234/v1` | 空欄でOK |
| Ollama | `http://localhost:11434/v1` | 空欄でOK |
| llama.cpp (server) | `http://localhost:8080/v1` | 空欄でOK |
| OpenAI | `https://api.openai.com/v1` | `sk-...` |

### 設定手順

1. サイドバーの **「設定」** を開きます。
2. **LLM接続** の欄でベースURL（必要ならAPIキー）を入力します。
3. **「モデル一覧を取得」** を押すと、接続先で使えるモデルが一覧に出ます。
4. 使いたいモデルを選び、**「このプロバイダを保存」** を押します。
5. **「接続テスト」** で `✔ 接続成功` が出れば準備完了です。

### 別のデバイスの LM Studio に繋ぐ（同一ネットワーク / VPN）

ベースURLに接続先のアドレスを入れます。**スキームや `/v1` は省略できます**
（`192.168.1.10:1234` と書けば `http://192.168.1.10:1234/v1` として扱われます）。

うまく繋がらない場合、以下の順に確認してください。

1. **接続先の LM Studio で「Serve on Local Network」を有効にする。**
   既定では `localhost` のみで待ち受けており、他のデバイスからは見えません。
2. **初回接続時に macOS が出す「ローカルネットワークへのアクセス」を許可する。**
   macOS 15 以降、同一ネットワーク上の機器へ接続するには利用者の許可が必要です。
   許可し忘れた場合は「システム設定 → プライバシーとセキュリティ → ローカルネットワーク」で
   EasyRoo を有効にしてください。
3. **接続先デバイスのファイアウォール**がそのポートを通しているか確認する。
4. **VPN 使用時**は、そのアドレスが VPN 経由で到達できるか
   （スプリットトンネルの除外対象になっていないか）確認する。
5. ホスト名（`mymac.local` など）で繋がらない場合は、**IPアドレスで直接指定**してみる。

> **接続テストは原因を教えてくれます**
> 「接続テスト」を押すと、失敗した理由（接続拒否・名前解決の失敗・到達不能・タイムアウト）と、
> それぞれに応じた確認事項が表示されます。単に「失敗しました」で終わりません。

> **モデル選びのコツ**
> ルーティーンではAIがツールを呼び出します。**ツール呼び出し（function calling）に対応したモデル**を
> 選んでください。ローカルなら Qwen3 Coder 系や GPT-OSS 系が扱いやすく、実績があります。

## 2.4 表示言語（日本語 / English）

左下の **「言語」** で切り替えます。

| 選択 | 動作 |
| --- | --- |
| **自動** | macOS の言語設定に追随します。既定値です |
| **日本語** | 常に日本語 |
| **English** | 常に英語 |

画面だけでなく、**AIへ渡す指示文も選んだ言語になります**。
手順書・注意書きをどちらの言語で書いても構いませんが、
表示言語と揃えたほうが指示の取りこぼしが減ります。

ターミナルからは `easyroo set '{"ui":{"language":"en"}}'` でも変更できます。

## 2.5 外観（ライト / ダーク）

左下の **「外観」** で表示テーマを切り替えられます。

| 選択 | 動作 |
| --- | --- |
| **自動** | macOS のシステム設定（ライト/ダーク）に追随します。既定値です |
| **ライト** | 常にライトテーマ |
| **ダーク** | 常にダークテーマ |

選択内容はすぐに反映され、次回起動時にも保たれます。
ターミナルからは `easyroo set '{"ui":{"theme":"dark"}}'` でも変更できます。

> 配色はライト・ダークとも、本文・補助テキスト・バッジのすべてが
> WCAG 2.1 AA のコントラスト比（本文 4.5:1／大きな文字 3:1）を満たすよう調整済みです。

## 3. 最初のルーティーンを作る

**「ルーティーン」→「＋ 新しいルーティーン」** を押すと、作成画面が開きます。

### 入力する項目

#### ルーティーン名
一覧に表示される名前です。例:「朝のニュース要約」

#### 目的
このルーティーンで何を達成したいか、1行で書きます。
例:「主要ニュースを集めて要約をデスクトップに保存する」

#### 手順書 ★重要
**AIがこの通りに実行します。** 番号付きで、具体的に書くほど安定します。

```
1. SearXNG (http://localhost:8888) で「AI ニュース」を検索する
2. 上位5件の見出しと要点を日本語でまとめる
3. ~/Desktop/news/ に YYYY-MM-DD.md というファイル名で保存する
4. 保存できたことを確認して終了する
```

#### 注意書き ★重要
**絶対に守らせるルールです。手順書より優先されます。**
ここに書いた内容は、AIが作業中に何を見ても曲げられません。

```
・ファイルの削除は絶対に行わない
・書き込みは ~/Desktop 配下のみ
・外部にデータを送信しない
・sudo は使わない
```

> 手順書やWebから取得した内容に「これを実行せよ」と書かれていても、
> AIはそれを**指示ではなくデータとして扱う**よう設定されています。
> 注意書きに反する操作を求められた場合、AIは実行せず理由を報告して終了します。

### 実行タイミングを決める

| 種類 | 説明 | 設定項目 |
| --- | --- | --- |
| **手動実行のみ** | 「今すぐ実行」を押した時だけ動く | — |
| **曜日を指定（毎週）** | 選んだ曜日の指定時刻に毎週動く | 曜日、時刻 |
| **日を指定（毎月）** | 指定した日の指定時刻に毎月動く | 日（1〜31）、時刻 |
| **一定間隔ごと** | 指定した分数ごとに繰り返す | 間隔（分） |

> 「毎月31日」のように該当しない月がある指定でも、次に31日が来る月まで正しく繰り越します。

### AIに許可する操作

- **ターミナル操作**: シェルコマンドの実行を許可するか
- **使用するMCPサーバ**: このルーティーンで使わせるMCPツールを選ぶ

### 詳細設定（省略可）
モデル、最大ステップ数、作業ディレクトリをルーティーンごとに上書きできます。
空欄なら「設定」画面の全体設定が使われます。

## 4. 動かす・止める

### 動かす

| 操作 | 効果 |
| --- | --- |
| **スタート** | スケジュールを有効化。以降、指定時刻に自動実行されます |
| **▶ 今すぐ実行** | スケジュールとは関係なく、その場で1回だけ実行します |

### 止める（3段階）

| 操作 | 効果 | 使う場面 |
| --- | --- | --- |
| **■ 即時停止** | 実行中のものだけを直ちに止める | 今動いている処理を止めたい |
| **スケジュール停止** | ルーティーン自体を無効化し、実行中なら停止 | このルーティーンをしばらく使わない |
| **緊急停止** | 全ルーティーンを停止 + スケジューラも一時停止 | とにかく全部止めたい |

**緊急停止** はサイドバー下部の赤いボタン、またはキーボードの **⌘⇧.**（Command+Shift+ピリオド）です。
LLMの応答待ちでも、シェルコマンドの実行中でも、その場で止まります。
実行中のコマンドは子プロセスまで含めて確実に終了します。

> 緊急停止するとスケジューラも止まります。再開するには
> 「ルーティーン」画面上部に出る **「スケジューラを再開」** を押してください。

### 実行の様子を見る

**「実行ログ」** を開くと、直近の実行が一覧で出ます。
クリックすると、AIが何を考え、どのコマンドを打ち、何が返ってきたかをステップごとに追えます。
実行中のログはリアルタイムで流れます。

## 4.4 実行が重なったとき

「毎時実行」のルーティーンが70分かかると、次の発火時刻に前回がまだ動いています。
このときの振る舞いを、ルーティーンごとに選べます（編集画面の **「実行が重なったとき」**）。

| 選択 | 動作 | 向いている処理 |
| --- | --- | --- |
| **見送る**（既定） | 実行中なら今回は動かしません | 二重に走ると困る処理 |
| **終わってから実行** | 完了後に1回だけ実行します | 取りこぼしたくない処理 |
| **止めてやり直す** | 実行中のものを停止して新しく始めます | 最新状態だけが意味を持つ処理 |

> 予約は**ルーティーンあたり1件まで**です。溜め込むと、遅延の原因が解消した後に
> 一斉に走り出して事態を悪化させるためです。

予約は緊急停止で破棄されます（止めたのに後から動き出すのは意図に反するため）。

```bash
easyroo update <id> --overlap queue   # skip | queue | restart
easyroo queued                        # 予約の一覧
easyroo unqueue <id>                  # 予約の取り消し
```

## 4.5 実測値を見る

サイドバーの **「実測値」** に、実行の集計が出ます。

| 指標 | 内容 |
| --- | --- |
| 実行回数 | 成功 / 失敗 / 停止の内訳つき |
| 成功率 | 全実行に占める成功の割合 |
| 平均・合計所要時間 | 1回あたりと累計 |
| トークン | 入力・出力・合計 |
| 拒否されたコマンド | 禁止規則で止めた回数 |
| ツール利用 | ツールごとの呼び出し回数・失敗数・平均所要時間 |

期間は **全期間 / 7日 / 30日** で切り替えられます。
ルーティーン別の内訳も同じ画面に出ます。

個々の実行については、「実行ログ」で実行をクリックし **「実測値」タブ** を開くと、
その実行だけのツール内訳と所要時間が見られます。

ターミナルからは:

```bash
easyroo stats                    # 全期間
easyroo stats --days 7           # 直近7日
easyroo stats --routine <id>     # 特定のルーティーンだけ
```

## 4.6 記憶（State / Journal）

ルーティーンは既定で、**実行をまたいで記憶を引き継ぎます**。
これにより「前回どこまで進んだか」「この環境の癖」といった知見が蓄積され、
毎回ゼロから始めずに済みます。

| ファイル | 役割 |
| --- | --- |
| **STATE.md** | 「いま」の状態。AIが `state_write` で丸ごと書き換えて維持します |
| **JOURNAL.md** | 実行ごとの記録。直近5件がAIに渡されます |

実行が始まると、この2つが自動でAIへ渡されます。
AIは作業の結果に応じて STATE を更新し、要点を JOURNAL に追記します
（追記し忘れても、実行の要約は自動で記録されます）。

- **STATE は追記ではなく置換**です。際限なく膨らまないようにするためです。
- 保存先は `~/Library/Application Support/EasyRoo/memory/<ルーティーンID>/` です。
  **ただの Markdown なので、Finder で開いて中身を読んだり直したりできます。**
  AIが何を覚えているかを人が検証できることを重視しています。

ルーティーン編集画面の **「記憶（State / Journal）」** で、
引き継ぎのオン/オフと、記憶の閲覧・編集・消去ができます。

ターミナルからは:

```bash
easyroo memory <id>                        # 記憶を表示
easyroo memory-set <id> --file ./state.md  # STATE を差し替え
easyroo memory-note <id> --text "メモ"     # 記録に追記
easyroo memory-clear <id>                  # 記憶を消去
```

### 長い実行でのコンテキスト管理

1回の実行が長引くと、会話履歴がモデルの扱える長さを超えて失敗します。
EasyRoo はこれを3段構えで防ぎます。

1. **予算で判定する** — モデルの窓から出力用・プロンプト用の余白を引いた分を
   履歴の予算とします。文字数固定ではなくトークン基準で、実際のAPI応答から
   1トークンあたりの文字数を**実行しながら較正**します。
2. **圧縮の前に記憶へ退避する** — 予算の手前に来たら、圧縮する前にAIへ
   「この先の作業に必要な事実だけを STATE に書き直せ」と指示します。
   要約は再要約で劣化しますが、**STATE に退避したものは劣化しません**。
3. **超過したら畳んで再試行する** — それでもモデルが「長すぎる」と返した場合、
   実行を失敗させずに履歴を畳んで1回だけやり直します。

これらは実行ログの「実測値」タブで、圧縮回数・記憶への退避回数として確認できます。

モデルの窓は「設定 → 実行の安全装置」で指定します（既定 32768 トークン）。
**実際の窓より少し小さめにしておくのが安全です。**

## 4.7 サブラン（部分作業の切り出し）

「20個のファイルを1件ずつ処理する」ような手順書では、
全ファイル分の作業ログが同じ文脈に積み上がり、後半で序盤が失われます。

**サブラン**を許可すると、AIは独立した部分作業を
**まっさらな文脈で動く子エージェント**に委任できます。
親には結果の要約だけが返るため、本体の文脈を消費しません。

- 子は親の**手順書・注意書き・禁止規則・作業ディレクトリを継承**します
- 子は**会話履歴を継承しません**（それがこの機構の目的です）
- **入れ子は1段まで**。子はさらに子を作れません
- 子は記憶ツールを持ちません。記憶の一貫性は親が責任を持ちます

ルーティーン編集画面の「AIに許可する操作」で切り替えます（既定は有効）。

```bash
easyroo update <id> --subrun false    # サブランを禁止する
```

## 5. MCPハブ

MCPサーバを登録すると、そのツールをAIがルーティーン内で呼び出せるようになります。

### サーバを追加する

**「MCPハブ」→「＋ サーバを追加」**

#### コマンド起動型（stdio）
ローカルで動かす一般的なMCPサーバです。

- **サーバ名**: `filesystem`（ツール名の接頭辞になります。英数字推奨）
- **コマンド**: `npx`
- **引数**: `-y @modelcontextprotocol/server-filesystem /Users/あなた/Documents`
- **環境変数**: APIキーが必要なサーバでは `{"API_KEY":"..."}` の形式で渡します

#### リモート型（HTTP）
- **URL**: `http://localhost:3000/mcp`

**「保存して接続」** を押すと接続され、取得できたツールの一覧がカードに表示されます。

### 使い方のポイント

- 接続したツールは `mcp__<サーバ名>__<ツール名>` という名前でAIに見えます。
  サーバ名で名前空間が分かれるため、複数サーバで同名ツールがあっても衝突しません。
- ルーティーン編集画面の **「使用するMCPサーバ」** で、
  どのルーティーンにどのサーバを使わせるかを個別に選べます。
- 1台のサーバが落ちても他のサーバやアプリ本体には影響しません。
  カードに「エラー」と理由が表示されます。
- **サーバが予期せず落ちた場合は自動で再接続します**（1秒 → 3秒 → 10秒 → 30秒 → 60秒と間隔を広げて再試行）。
  無効化したサーバは追いかけません。
- **実行開始時、必要なMCPサーバの接続完了を待ちます**（既定30秒）。
  起動直後のスケジュール実行がツール無しで走ってしまうのを防ぎます。
  待っても繋がらないサーバがある場合は、実行ログにその旨が残ります。

## 6. ターミナルからの操作（AI向け）

EasyRoo は **ローカル制御API** を内蔵しており、`easyroo` コマンドで
**GUIでできることをすべてターミナルから実行できます**。
Claude Code などのAIエージェントに、EasyRoo自体の設定・管理・実行を任せられます。

### インストール

「設定」画面の **「easyroo コマンドをインストール」** を押します。
権限が足りない場合は、画面に表示されるコマンドをターミナルに貼り付けて実行してください。

> APIは `127.0.0.1` のみで待ち受け、Bearerトークン必須です。
> トークンは `~/Library/Application Support/EasyRoo/runtime.json` に
> 所有者のみ読める権限(600)で保存され、`easyroo` コマンドが自動で読み込みます。
> 外部のネットワークからは一切アクセスできません。

### よく使うコマンド

```bash
# 状態を見る
easyroo status                  # アプリ全体の状態
easyroo list                    # ルーティーン一覧

# ルーティーンを作る
easyroo create --name "朝のニュース要約" \
  --goal "主要ニュースを集めて要約をデスクトップに保存する" \
  --procedure-file ./procedure.md \
  --constraints "ファイルの削除は絶対に行わない。書き込みは ~/Desktop 配下のみ。" \
  --schedule weekly --weekdays 1,2,3,4,5 --time 07:30

# 動かす・止める
easyroo start <id>              # スケジュール有効化（スタートボタン相当）
easyroo run <id> --wait         # 今すぐ実行して完了まで待つ
easyroo kill <id>               # 実行中のものを即時停止
easyroo stop <id>               # スケジュールごと停止
easyroo emergency-stop          # 全停止 + スケジューラ停止
easyroo resume                  # スケジューラ再開

# ログを見る
easyroo runs                    # 実行履歴
easyroo log <runId>             # 実行の詳細ログ（読みやすく整形）

# 実測値
easyroo stats                   # 実行回数・成功率・所要時間・ツール別内訳
easyroo stats --days 7          # 直近7日だけ

# 実行の予約
easyroo queued                  # 完了後に実行される予約
easyroo unqueue <id>            # 予約を取り消す

# 記憶
easyroo memory <id>             # 引き継いでいる記憶を表示
easyroo memory-set <id> --file ./state.md
easyroo memory-note <id> --text "気づいたこと"
easyroo memory-clear <id>

# 禁止コマンド
easyroo deny-categories         # カテゴリ一覧
easyroo deny-check 'rm -rf /'   # 通るか試す

# 設定
easyroo models                  # 使えるモデル一覧
easyroo set '{"maxSteps":40}'   # 設定をJSONで更新
easyroo test-provider           # LLM接続テスト

# MCP
easyroo mcp                     # サーバ一覧と接続状態
easyroo mcp-add --name fs --command npx \
  --args "-y,@modelcontextprotocol/server-filesystem,/Users/me/Documents"
easyroo mcp-call <id> --tool read_file --args '{"path":"/tmp/a.txt"}'
```

すべてのオプションは `easyroo help` で確認できます。

### AIエージェントに任せる例

```
EasyRoo で「毎週月曜9時に、先週のGit活動をまとめてレポートを作る」
ルーティーンを作って、スタートしておいて。
注意書きには、リポジトリへの書き込みは一切しないことを入れて。
```

とAIに伝えれば、AIは `easyroo create` → `easyroo start` を実行して設定を完了できます。

## 7. 安全に使うために

EasyRoo はAIにターミナルを渡すアプリです。以下の安全装置を用意しています。

| 装置 | 内容 | 設定場所 |
| --- | --- | --- |
| **注意書き** | ルーティーンごとの絶対厳守ルール | ルーティーン編集 |
| **禁止コマンド（8カテゴリ）** | 該当するコマンドは実行前に拒否 | 設定 / ルーティーン編集 |
| **作業ディレクトリ** | AIの作業場所を限定 | 設定 / ルーティーン編集 |
| **最大ステップ数** | 暴走時の打ち切り（既定30） | 設定 |
| **実行全体の上限時間** | 既定30分で強制終了 | 設定 |
| **1コマンドの上限時間** | 既定120秒で強制終了 | 設定 |
| **ターミナル操作の無効化** | シェル実行そのものを禁止できる | 設定 / ルーティーン編集 |
| **多重実行の防止** | 同じルーティーンは同時に1つまで | 自動 |

### 禁止コマンドのカテゴリ

既定で以下の8カテゴリが有効です。カテゴリ単位でオン/オフでき、
**ルーティーンごとに上書き**もできます。

| カテゴリ | 例 |
| --- | --- |
| **システム破壊・大量削除** | `rm -rf`（特に `/`・`~`・`/System`・`/Library`・`/Applications` 配下）、`find … -delete`、`diskutil eraseDisk`、`diskutil partitionDisk`、`newfs_*`、`mkfs*`、`dd of=…`、`gpt destroy` |
| **ネットから取得したコードの即実行** | `curl … \| sh`、`curl … \| bash`、`wget … \| sh`、`bash <(curl …)`、`eval` + `curl` |
| **権限の一括変更** | `chmod -R 777`、`chmod 000`、`chown -R`、`sudo chown -R` |
| **Gitの破壊操作** | `git reset --hard`、`git clean -fdx`、`git push --force`、`git rebase --onto`、`git filter-repo` |
| **システム制御** | `shutdown`、`reboot`、`halt`、`launchctl bootout`、`launchctl unload`、`csrutil disable` |
| **ネットワーク**（条件付き） | `ssh`、`scp`、`rsync`、`nc`、`curl`、`wget` — **信頼ドメイン宛のみ許可** |
| **パッケージ管理** | `brew uninstall`、`brew cleanup`、`brew services stop` |
| **権限昇格** | `sudo`、`su`、`doas` |

#### ネットワークの条件付き許可

ネットワーク系のコマンドは、宛先が **信頼ドメイン** に含まれる場合だけ通ります。
既定の信頼ドメインは以下です（設定画面で追加・削除できます）。

```
localhost / 127.0.0.1 / ::1
github.com / raw.githubusercontent.com / api.github.com
codeload.github.com / objects.githubusercontent.com
```

つまり `curl http://localhost:8888/…`（SearXNG など）や
GitHub からの取得は通り、見知らぬドメインへの通信は止まります。

#### 動作を試す

設定画面の禁止コマンド欄に、試したいコマンドを入れて確認できます。
ターミナルからも試せます。

```bash
easyroo deny-categories                       # カテゴリ一覧
easyroo deny-check 'rm -rf /'                 # → 拒否されます
easyroo deny-check 'curl http://localhost/x'  # → 許可されます
```

#### ルーティーンごとの調整

ルーティーン編集画面の **「禁止コマンド」** で、
「全体設定を引き継ぐ」を外すとそのルーティーン専用の設定になります。
引き継いだまま、追加の禁止パターンや信頼ドメインを足すこともできます。

```bash
easyroo update <id> --deny-categories destructive,privilege
easyroo update <id> --deny-add '\bmy-dangerous-tool\b'
easyroo update <id> --trust internal.example.com
```

#### 追加パターンの書き方（正規表現）

追加の禁止パターン・許可パターンは正規表現として扱われます。
ただし、次のものは**安全のため採用されません**。

- 400文字を超えるもの
- `(a+)+` のように、繰り返しの中にさらに繰り返しが入る形
  （判定に非常に長い時間がかかり、アプリが固まる原因になるため）

採用されなかったパターンは、拒否側なら「効かない」、許可側なら「穴を開けない」
という安全な側に倒れます。設定画面の確認欄でコマンドを試すと、
`⚠ deny: "…" — nested quantifier (ReDoS risk)` のように理由が表示されるので、
書き直してください。

> `rm -rf /` のような**そのままの文字列**を書いた場合は、
> 正規表現として不正でもそのまま文字列一致として扱われます。難しく考える必要はありません。

> AIがコマンドを拒否されると、その旨がツールの結果として返され、
> 「回避せず別の手段を検討するか finish で報告する」よう指示されています。
> 拒否された回数は実測値に記録されます。

### 推奨する使い方

1. **最初は「手動実行のみ」で作り、`▶ 今すぐ実行` で動きを確認してから** スケジュールを設定する。
2. **注意書きには「してはいけないこと」を具体的に書く。**
   「気をつける」ではなく「〜は絶対に行わない」と書くほど確実に効きます。
3. **作業ディレクトリを専用フォルダに限定する。**
4. 既定の禁止パターン（`rm -rf /`、`sudo `、`mkfs` など）は消さず、必要に応じて追加する。

## 8. データの保存場所

```
~/Library/Application Support/EasyRoo/
├── settings.json     設定（LLM接続、安全装置、ターミナル設定）
├── routines.json     ルーティーンの定義
├── mcp.json          MCPサーバの登録内容
├── runtime.json      制御APIのポートとトークン（権限600）
├── runs/             実行ログ（最新200件を保持。実測値の集計元）
├── memory/           ルーティーンごとの記憶
│   └── <id>/
│       ├── STATE.md     現在の状態
│       └── JOURNAL.md   実行ごとの記録
├── logs/             アプリ内部のエラーログ
└── workspace/        既定の作業ディレクトリ
```

「設定」画面の **「データフォルダを開く」** からFinderで開けます。
バックアップはこのフォルダごとコピーすれば完了です。

## 9. 困ったときは

| 症状 | 対処 |
| --- | --- |
| 「LLMプロバイダが設定されていません」 | 設定画面でベースURLとモデルを保存してください |
| 接続テストが失敗する | 表示される原因と確認事項に従ってください。別デバイスへ繋ぐ場合は「2. 事前準備」の手順を参照 |
| 別のPCのLM Studioに繋がらない | ①接続先で「Serve on Local Network」を有効化 ②macOSの「ローカルネットワーク」許可 ③ファイアウォール ④VPNの経路 の順に確認 |
| 「接続成功」だがモデルが0件 | ベースURLが誤っている可能性があります（v1.0で、OpenAI互換でない応答は失敗として扱うよう修正済み） |
| MCPサーバが落ちたまま戻らない | v1.0で自動再接続するようになりました。バックオフしながら復帰を試みます |
| AIがツールを使わず喋るだけで終わる | ツール呼び出し対応モデルに変更してください |
| 「最大ステップ数に達した」で終わる | 手順書をより具体的に分割するか、最大ステップ数を増やす |
| 時刻になっても動かない | ルーティーンが「有効」か、緊急停止でスケジューラが止まっていないか確認 |
| MCPサーバが「エラー」になる | カードに出ている理由を確認。コマンドのパスや引数を見直してください |
| `easyroo` が「接続できません」と言う | EasyRoo.app が起動しているか確認してください |
| 画面が見づらい／まぶしい | 左下の「外観」でライト・ダークを切り替えてください |
| 表示を英語にしたい | 左下の「言語」で English を選んでください |
| AIがコマンドを実行できない | 禁止規則に当たっている可能性があります。`easyroo deny-check '<コマンド>'` で確認し、必要ならカテゴリを外すか信頼ドメインを足してください |
| 前回の続きから始めてほしい | ルーティーン編集の「記憶」を有効にし、手順書に「STATEを確認してから進める」と書いてください |
| 長い実行が途中で失敗する | 「設定 → モデルのコンテキスト長」が実際の窓と合っているか確認してください。手順書の分割やサブランの活用も有効です |
| 毎回の発火が飛ばされる | 前回が終わっていません。編集画面の「実行が重なったとき」で「終わってから実行」を選ぶと取りこぼしません |
| 文脈がすぐ一杯になる | 記憶を有効にすると、圧縮前に STATE へ退避されます。独立した部分作業はサブランに逃がしてください |

ウィンドウを閉じてもスケジューラは動き続けます（macOSの慣習に合わせた挙動です）。
完全に終了するにはメニューから **「EasyRoo を終了」** を選んでください。

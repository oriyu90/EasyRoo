## 3. モジュールの地図

処理の中心は `engine.js`。**GUI（IPC）も CLI（HTTP API）も、必ず Engine の同じメソッドを呼ぶ。**
この一本化が「AIがターミナルから全操作できる」という要件を構造で保証しているので、
機能追加時もこの原則を崩さないこと。

| ファイル | 行数 | 役割 |
| --- | --- | --- |
| `main/engine.js` | 436 | **中核**。全操作の唯一の窓口。実行制御・重複ポリシー・予約 |
| `main/runner.js` | 895 | 実行エンジン。エージェントループ、圧縮、記憶退避、サブラン |
| `main/store.js` | 275 | JSON永続化（アトミック書き込み）と設定の正規化 |
| `main/scheduler.js` | 163 | 20秒 tick でのスケジュール判定 |
| `main/api.js` | 239 | ローカル制御API（127.0.0.1 + Bearer） |
| `main/index.js` | 308 | Electron エントリ、IPC、ウィンドウ、CLI設置 |
| `main/llm.js` | 143 | OpenAI互換クライアント |
| `main/netdiag.js` | 213 | URL正規化と接続失敗の原因判定 |
| `main/contextBudget.js` | 131 | コンテキスト予算とトークン較正 |
| `main/memory.js` | 195 | 長期記憶（STATE.md / JOURNAL.md） |
| `main/metrics.js` | 163 | 実測値の集計（参照時に計算） |
| `main/mcp/hub.js` | 236 | MCPハブ。接続管理・自動再接続・接続待ち |
| `main/mcp/client.js` | 270 | MCPクライアント（stdio / Streamable HTTP）自前実装 |
| `main/tools/builtin.js` | 372 | 組み込みツール定義と実行 |
| `main/tools/denyRules.js` | 303 | 禁止コマンドの規則エンジン（8カテゴリ） |
| `shared/i18n.js` | 697 | 多言語辞書（UMD。main / CLI / renderer で共用） |
| `renderer/` | — | GUI。**ビルド工程なし**。素のDOM操作 |
| `bin/easyroo.js` | — | CLI。`runtime.json` から接続先を自動取得 |

### 手を入れるときの約束

これらは設計の前提であり、破ると別の場所が壊れる。

1. **実行時依存を増やさない。** 安定性の根拠がこれ。追加が必要なら、まず自前実装を検討する
2. **レンダラにビルド工程を導入しない。** 素のDOM + UMD の i18n で完結させている
3. **`app.css` に生の色・書体を書かない。** すべて `tokens.css` のトークンを参照する
4. **GUI に機能を足したら、Engine 経由にする。** IPC と HTTP API の両方から使えることが要件
5. **1実行 = 1 AbortController。** 新しい非同期処理は必ず signal を伝播させる
6. **assistant(tool_calls) と tool メッセージは必ず組で扱う。** 分離すると API が 400 を返す

## 5. 検証時に踏んだ落とし穴

**同じ時間を溶かさないために、実際に踏んだものだけを挙げる。**

### 5.1 古いDMGボリュームが残り、古いビルドを検証してしまう ★

`hdiutil attach` を繰り返すと `/Volumes/EasyRoo 1.0.0`, `... 1`, `... 2` が同時に存在し、
`cp -R "/Volumes/EasyRoo 1.0.0/EasyRoo.app"` が**古い方**を掴む。
修正が効いていないように見え、原因追及に時間を溶かした。

```bash
# 必ずマウント先を出力から取得する
MP=$(hdiutil attach xxx.dmg -nobrowse | grep -o '/Volumes/.*$' | tail -1)
cp -R "$MP/EasyRoo.app" /tmp/検証先/
hdiutil detach "$MP" -quiet
# 事前に残骸を掃除する
for v in /Volumes/EasyRoo*; do [ -d "$v" ] && hdiutil detach "$v" -force -quiet; done
```

疑わしいときは asar の中身を直接見る:
```bash
npx asar extract-file <app>/Contents/Resources/app.asar src/main/llm.js
```

### 5.2 単一インスタンスロックで、2つ目が黙って終了する

`app.requestSingleInstanceLock()` により、既に EasyRoo が動いていると
新しいプロセスは即座に終了する。`runtime.json` が作られないため
「起動しない」「クラッシュした」ように見える。

```bash
pkill -f "EasyRoo.app/Contents/MacOS/EasyRoo"; sleep 2
lsof -nP -iTCP:8787 -sTCP:LISTEN   # 掴んでいるプロセスの確認
```

### 5.3 localStorage は EASYROO_HOME と独立

テーマ・言語の即時反映用の控えは Electron の userData に保存される。
`EASYROO_HOME` を変えても**リセットされない**ため、
初期状態の検証時に前回の選択が残る。

### 5.4 Browser pane は `file://` を静的スナップショットとして描画する

UI検証でクリックが効かない。HTTP 経由にすれば通常どおり動く。

```bash
cd easyroo && python3 -m http.server 8899
# → http://localhost:8899/test/ui-preview.html
```

### 5.5 ブラウザ側の非同期処理はツールがタイムアウトしても走り続ける

30秒でタイムアウトしても promise は生きており、画面を切り替え続ける。
以降の測定が別画面の値を拾って混乱する。**長い監査は分割して呼ぶこと。**

### 5.6 LAN到達性は同一マシンからは再現できない

自機の LAN IP 宛の接続は macOS のローカルネットワーク権限を発火させない。
`netdiag.js` の分類ロジックは**エラーコードを注入する単体テスト**で検証している
（実機の別デバイスが必要な部分は、そもそもテストで担保できない）。

### 5.7 ESLint は飾りではない

型チェックの無い構成のため、静的解析が実バグを検出している。

- `no-unused-vars` → **サブランが親の全ツールを継承**していた（入れ子1段の制限が破れる）
- `no-dupe-keys` → 多言語辞書の英語側に日本語が混入（一括置換の副作用）

`npm run check` を必ず通すこと。

### 5.8 一括置換（`str.replace`）は全件置換される

Python の `str.replace` は既定で全出現を置換する。
i18n 辞書のように日英で同じキー名が並ぶファイルでは、
**日本語ブロックの内容が英語ブロックにも入る**事故が起きた（5.7 の後者）。
辞書を編集したら `npm run lint` で重複キーを確認すること。

### 5.9 自動バグスキャンの指摘は、必ず実地で裏を取る ★

第6回のスキャンは「レンダラにイベントリスナの解除漏れが20箇所以上ある」と
報告したが、確認すると**不具合ではなかった**。該当リスナはすべて `render()` が
毎回作り直すノードに張られており、`innerHTML = ''` でノードごと捨てるため
一緒に回収される。差分レンダリング前提の一般論を、差分を持たない本アプリに
当てはめた誤検出だった。

指摘どおりに `removeEventListener` を足すと、コード量が増えるうえに
「毎回全部作り直すから壊れない」という設計の根拠まで薄まる。
**指摘を消化するために設計を曲げないこと。**
判断の記録は設計書 8.5 に残してある。

一方、同じスキャンが挙げた ReDoS とポーリングの停止漏れは本物だった。
**当たりと外れが混ざって出てくる**前提で、1件ずつ次を確かめる。

1. 実際にその経路が通るか（呼び出し元をたどる）
2. 起きたとして何が壊れるか（固まる／消える／漏れる）
3. 再現するテストを書けるか（書けないなら、まだ理解できていない）

## 6. 未解決事項

### 6.1 アイコン元図案の出所（★ 要判断・3セッション未解決）

`assets/source-icon.ico`（元ファイル名 `IMG_0147.ico`）は利用者提供だが、**作成者情報・出典のメタデータが無く**、
技術的に出所を確認できなかった（AirDrop 経由の痕跡のみ）。

MIT で再頒布する前に確認が必要。特に:

- **Apple SF Symbols** — ライセンス上、**アプリのアイコンやロゴとしての使用は明確に禁止**
- **Font Awesome / Material Icons など** — CC BY 4.0 や Apache-2.0 が多く、帰属表示が必要な場合がある

自作であれば対応不要。差し替える場合は
`assets/make-icon.js` の `GLYPH` 定数（元絵256px座標系での採寸値）を直して再生成する:

```bash
npm run icon     # icon.png と icon.icns をまとめて再生成
```

生成は決定的で、同じ入力からは**バイト単位で同一の icns** が出る（確認済み）。
採寸のやり方はスクリプト冒頭のコメントに書いてある。

### 6.2 コード署名がない

`identity: null` でビルドしており、初回起動で Gatekeeper の警告が出る。
回避手順は README に記載済み。Apple Developer ID があれば署名し直せる。

### 6.3 Intel Mac 向けビルドは未検証

`--arm64` のみでビルドしている。`--x64` を追加すれば作れるが、動作確認はしていない。
